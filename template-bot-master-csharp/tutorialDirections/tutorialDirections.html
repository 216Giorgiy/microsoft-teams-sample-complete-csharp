Directions

Register a new bot (or update an existing one) with Bot Framework by using your BASE_URI + /api/messages (e.g. https://####abcd.ngrok.io/api/messages) as the bot's "Messaging endpoint". Bot registration is Bot Registration. NOTE: When you create your bot you will create an App ID and App password - make sure you keep these for later.

Once registered, take your new App ID and App password and put these values into the Web.config file as the MicrosoftAppId and MicrosoftAppPassword respectively. Do this by replacing the 'NeedToSetThis' for each. All of your Web.config values should now be filled.

This App ID is now the value you should enter for 'REGISTERED_BOT_ID' in the manifest. It is also the value you should use at the top level 'id' value instead of the original temporary ID from before. Thus, all app.ids refer to this registered App ID.

File Changes

  • controllers/MessagesController.cs - used to accept the incoming bot messages

  • bot/dialogs/RootDialog.cs - this file is used to accept all incoming messages and send them to the correct dialog

  • bot/dialogs/HelloDialog.cs - this dialog is used to respond with 'Hello!' when 'hello' is entered

  • bot/dialogs/DefaultDialog.cs - this dialog is used handle unrecognized commands

  • bot/middleware/StripBotAtMentions.cs - this middleware is used to strip the '@botName' from messages that come from channels so the commands can be parsed correctly

  • App_Start/WebApiConfig.cs - added to route the messages to the controller MessagesController

  • Global.asax - added to call App_Start/WebApiConfig.cs at startup

  • Changed the manifest to add the bot to the app.

Setup and Sideload the Manifest

  • In the manifest replace all instances of '<<BASE_URI>>' with the https BASE_URI endpoint you received from ngrok.

  • In the manifest replace all instances of '<<BASE_URI_DOMAIN>>' with the https BASE_URI endpoint you received from ngrok WITHOUT the 'https://' part e.g. ####abcd.ngrok.io

  • In the manifest replace all instances of '<<REGISTERED_BOT_ID>>' with the your REGISTERED_BOT_ID (MicrosoftAppId in Web.config)
